KVCache Storage

Goal

Unified management of KVCache on DRAM/SSD, as an independent daemon, decoupled from inference frameworks (vLLM/sglang/MiniMax internal framework), while reporting KVCache metadata to scheduler for KVCache-aware scheduling.

Core Features

  • Unified management of KVCache on DRAM + SSD
  • Independent daemon, decoupled from inference frameworks
  • KVCache metadata reporting to scheduler for KVCache-aware scheduling

Architecture

  • Independent Daemon: KVCache storage as independent component, not dependent on inference framework
  • Multi-tier Storage: Supports DRAM and SSD two-tier KVCache storage
  • Metadata Reporting: Reports KVCache metadata to scheduler
  • Framework Compatibility: vLLM, Sglang, MiniMax internal framework

Results

  • Saved thousands of GPUs
  • Supports multiple inference frameworks
  • Enables KVCache-aware scheduling
